# CV Pipeline — Real-Time Student Engagement Detection

A computer vision microservice that analyzes a student's webcam feed in real time to measure **attention**, **fatigue**, and **boredom**. Built as part of the DALI educational platform.

## Overview

During an online class, the student's browser captures webcam frames and sends them to this service. For each frame, the service extracts facial landmarks, eye metrics, head pose, and runs a deep learning model to produce three scores:

| Score | What it measures | Range |
|---|---|---|
| **Attention** | Is the student looking at the screen? | 0.0 (distracted) → 1.0 (focused) |
| **Fatigue** | Is the student drowsy or falling asleep? | 0.0 (alert) → 1.0 (drowsy) |
| **Boredom** | Is the student emotionally disengaged? | 0.0 (engaged) → 1.0 (bored) |

These scores are stored in MongoDB and can be aggregated per student or per class over a rolling time window, allowing a teacher dashboard to monitor classroom engagement in real time.

```
┌─────────────────┐      POST /cv/frame       ┌──────────────────────┐
│  Student client │  ───────────────────────► │  CV Service          │
│  (browser + JS) │  ◄─────────────────────── │  FastAPI :8005       │
└─────────────────┘    {attention, fatigue,   └──────────┬───────────┘
                        boredom, ...}                    │
                                                         │ insert event
                                                         ▼
┌─────────────────┐      GET /cv/aggregate    ┌──────────────────────┐
│ Teacher dash    │  ───────────────────────► │  MongoDB             │
│ (browser + JS)  │  ◄─────────────────────── │  cv_events coll.     │
└─────────────────┘    rolling 60s averages   └──────────────────────┘
```

The service uses a **two-engine design** — a rule-based geometric engine and a learned CNN engine — that run independently on every frame and are fused into final scores. If the CNN model fails to load, the rule engine still provides attention and fatigue scores, making the system robust.

## How It Works

### Engine 1 — Rule-Based (Geometric)

Uses **MediaPipe FaceMesh** (478 3D facial landmarks) to compute geometric features directly from the face, with no training data required:

| Feature | How it's computed | What it tells us |
|---|---|---|
| **EAR** (Eye Aspect Ratio) | Ratio of vertical to horizontal eye distances | Eye openness — low EAR = closed eyes |
| **PERCLOS** | % of frames with EAR below threshold over last 60 seconds | Sustained drowsiness indicator |
| **Gaze Score** | Distance from iris center to eye center | Whether the student is looking at the screen |
| **Head Pose** (Roll, Pitch, Yaw) | Perspective-n-Point (PnP) solver on 3D landmarks | Head orientation — turned away = not paying attention |

**Outputs:** Attention score + Fatigue score.

### Engine 2 — Learned (CNN Model)

A **MobileNetV2** convolutional neural network, fine-tuned on the [DAiSEE dataset](https://iith.ac.in/~daisee/) (9,068 student webcam video clips labeled for engagement). The model takes a 224×224 RGB face crop and outputs 4 softmax probabilities:

| Index | Class | Description |
|---|---|---|
| 0 | Boredom | Student appears bored |
| 1 | Confusion | Student appears confused |
| 2 | **Engagement** | Student appears engaged |
| 3 | Frustration | Student appears frustrated |

The model is exported to ONNX format (~9 MB) for fast CPU inference using `onnxruntime`.

**Output:** Boredom score (derived from the engagement probability).

### Score Fusion

The two engines are independent. Their outputs are combined using weighted formulas (see below). If the CNN model is unavailable, the system gracefully degrades: attention and fatigue still work, boredom defaults to 0 with a `model_fallback=true` flag.

---

## Scoring Formulas

```
Attention = 0.6 × gaze_attention  +  0.4 × head_attention
Fatigue   = 0.6 × PERCLOS         +  0.4 × (1 − ear_normalized)
Boredom   = 1 − engagement_confidence    (or 0 if model unavailable)
```

All scores are clamped to **[0, 1]**.

**Why 60/40 weighting?**
- For **attention**: gaze is the stronger signal (a student can face forward but look at their phone), so it leads.
- For **fatigue**: PERCLOS is the literature-validated metric; instantaneous EAR fluctuates with natural blinks, so it weighs less.

## Data Flow

For each webcam frame received via `POST /cv/frame`:

```
1. Decode base64 JPEG → BGR frame
2. Run MediaPipe FaceMesh → 478 landmarks
3. Compute EAR, gaze, head pose from landmarks (Engine 1)
4. Crop face region using landmarks
5. Feed face crop to MobileNetV2 ONNX model → engagement probability (Engine 2)
6. Update per-student PERCLOS rolling window
7. Fuse all signals → attention, fatigue, boredom
8. Store event document in MongoDB
9. Return scores to client
```

## API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/cv/frame` | Process a single webcam frame. Accepts `{meeting_id, student_id, frame_base64}`. Returns attention, fatigue, boredom scores. |
| `GET` | `/cv/aggregate/{meeting_id}/{student_id}` | Rolling 60-second average scores for one student. |
| `GET` | `/cv/aggregate/{meeting_id}/class` | Class-wide average scores across all students in a meeting. |
| `GET` | `/cv/health` | Health check. Returns the active model backend. |

## Training Pipeline (Notebook)

The file `notebooks/CV_DALI.ipynb` is designed to run on **Google Colab** with a T4 GPU. It performs:

1. **Dataset download** — Downloads and extracts the DAiSEE dataset (~6 GB of student webcam clips).
2. **Label parsing** — Reads engagement labels from CSV files for each video clip.
3. **Face extraction** — Uses MediaPipe FaceMesh to crop faces from video frames (matching the production inference pipeline exactly).
4. **Data augmentation** — Random flips, rotations, brightness/contrast adjustments.
5. **Model training** — Fine-tunes a pre-trained MobileNetV2 (ImageNet weights) with the top 30 layers unfrozen.
6. **ONNX export** — Converts the trained Keras model to `cv_model.onnx` for lightweight CPU inference.
7. **Parity verification** — Confirms Keras and ONNX outputs match within 1e-4.

After training, download `cv_model.onnx` from Colab and place it in the project root directory.

## Setup & Installation

### Prerequisites

- Python 3.10–3.12
- MongoDB (local or Atlas)
- A webcam (for local demo)

### Install Dependencies

```bash
pip install -r requirements.txt
```

### Environment Variables

Copy `.env.example` to `.env` and configure:

```bash
MONGO_URI=mongodb://localhost:27017
MONGO_DB=edu_platform
CV_PORT=8005
```

### Model File

The trained model (`cv_model.onnx`) is not included in the repository due to its size. To generate it:

1. Open `notebooks/CV_DALI.ipynb` in Google Colab.
2. Select **Runtime → Change runtime type → T4 GPU**.
3. Click **Runtime → Run All**.
4. Download the generated `cv_model.onnx` from the Colab file explorer.
5. Place it in the project root: `CV/cv_model.onnx`.

> **Note:** The service works without the model file — attention and fatigue will still be computed. Only boredom requires the trained model.

## Running the Application

### Option 1: FastAPI Server (Production)

```bash
python cv_pipeline.py
# Server starts at http://localhost:8005
# API docs at  http://localhost:8005/docs
```

### Option 2: Local Webcam Demo (Development / Testing)

```bash
python tools/live_demo.py --mode 3
```

Opens a real-time webcam window with overlaid scores. No server or database required.

**Keyboard controls:**
| Key | Action |
|---|---|
| `1` | Show Engine 1 only (rules → attention, fatigue) |
| `2` | Show Engine 2 only (model → boredom) |
| `3` | Show both engines fused |
| `l` | Toggle face landmark overlay |
| `r` | Reset PERCLOS history |
| `q` | Quit |

## Configuration

All tunable parameters are centralized in `app/config.py`:

| Parameter | Default | Description |
|---|---|---|
| `EAR_THRESHOLD` | 0.25 | EAR below this = eye closed |
| `EAR_NORMALIZED_MAX` | 0.35 | EAR of a fully open eye |
| `PERCLOS_WINDOW_SEC` | 60.0 | Rolling window for PERCLOS (seconds) |
| `ROLL_THRESH_DEG` | 30.0 | Max head roll before attention = 0 |
| `PITCH_THRESH_DEG` | 25.0 | Max head pitch before attention = 0 |
| `YAW_THRESH_DEG` | 25.0 | Max head yaw before attention = 0 |
| `GAZE_THRESH` | 0.4 | Max gaze offset before attention = 0 |
| `AGG_WINDOW_SEC` | 60.0 | Aggregation query window (seconds) |
